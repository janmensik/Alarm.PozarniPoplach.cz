## 2024-05-24 - [CSRF and XSS on Device Activation Page]
**Vulnerability:** Found a lack of Cross-Site Request Forgery (CSRF) protection on the device activation page (`view/page/activate.php`), which processes state-changing `POST` requests. Additionally, a minor Cross-Site Scripting (XSS) vulnerability was identified where `$device_code` was echoed without escaping in the form action URL (`tpl/page.activate.html`).
**Learning:** The application does not use a central framework that provides automatic CSRF protection for all forms. CSRF tokens must be manually generated, validated, and included in the Smarty templates for every form that performs state-changing actions.
**Prevention:** Always implement CSRF protection using `random_bytes` and `hash_equals` (or equivalent standard library functions) for state-changing endpoints. Furthermore, consistently escape variables using the `|escape` modifier in Smarty templates when reflecting user input or variables into HTML contexts, such as URLs or input values.
